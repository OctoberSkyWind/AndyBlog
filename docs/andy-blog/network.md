# network

#### 短连接，长连接，段轮询，长轮询，SSE，WebSocket
> 短连接指 TCP 连接完成一次请求 / 响应后立即断开，不进行连接复用；
>
> 长连接则是 TCP 连接建立后保持不断，供多次请求复用，HTTP/1.1 默认开启的 keep‑alive 即属于长连接机制。其中，SSE 是基于 HTTP 协议的单向、服务端主动推送的长连接通信，仅支持服务端向客户端推送数据；WebSocket 先通过 HTTP 协议完成握手升级，随后基于 TCP 实现全双工双向长连接，客户端与服务端可随时相互发送消息，是实时聊天室等强实时场景的主流方案。
>
> 短轮询是客户端定时发起 HTTP 请求，服务端立即响应，请求频繁、网络开销较大；
>
> 长轮询是客户端发送请求后，服务端挂起等待数据或超时后再返回，本质仍为请求‑响应模式，并非真正的全双工通信。长轮询实时性与资源效率通常弱于 WebSocket，而在浏览器兼容、跨网络环境、仅需服务端单向推送等场景下，SSE 往往比长轮询更简洁高效；长轮询仅在需严格兼容普通 HTTP 链路、不便使用 SSE 流式响应等特殊场景下有一定适用性，并非整体优于 SSE。
